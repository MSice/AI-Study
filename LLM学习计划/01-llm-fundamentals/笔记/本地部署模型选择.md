# 根据电脑配置选择本地部署模型

## 核心公式

> **模型占用显存 ≈ 参数量 × 每个参数占的字节数**

| 精度 | 每参数字节 | 说明 |
|------|-----------|------|
| FP16（半精度） | 2 字节 | 标准推理精度 |
| INT8（8位量化） | 1 字节 | 质量略降，速度快一倍 |
| INT4（4位量化） | 0.5 字节 | 质量再降一点，能跑更大模型 |

以 7B 模型为例：FP16 需要 14GB（跑不了）→ INT4 只需 3.5GB（轻松跑）。

## 配置一：RTX 3070（8GB 显存）+ 16GB 内存 + 2TB 机械

### LLM 文本模型（甜点区：7B + INT4）

| 模型 | 量化 | 显存占用 | 特点 | 推荐度 |
|------|------|---------|------|--------|
| Qwen2.5-7B-Q4 | INT4 | ~4.5GB | 中文很强，日常够用 | ⭐⭐⭐⭐⭐ |
| Llama3.1-8B-Q4 | INT4 | ~5GB | 英文强，中文一般 | ⭐⭐⭐⭐ |
| DeepSeek-R1-7B-Q4 | INT4 | ~4.5GB | 推理/代码强 | ⭐⭐⭐⭐ |
| Qwen2.5-3B-Q8 | INT8 | ~3.5GB | 速度快，能力有限 | ⭐⭐⭐ |
| Phi-3-mini-3.8B-Q4 | INT4 | ~2.5GB | 微软出品，小而精 | ⭐⭐⭐ |

### 硬件注意事项

- **机械硬盘**：不影响推理速度，只是首次加载模型慢（30秒~1分钟）。建议加 SSD
- **16GB 内存**：跑 7B 够用，运行时尽量关掉不必要的程序

## 配置二：RTX 5070Ti（12GB 显存）+ 32GB 内存 + 2TB SSD

> 12GB 显存是关键分水岭，很多"上一档"的模型刚好能塞进来。

### LLM 文本模型（甜点区升级：14B + INT4）

### 两套配置对比

| 能力 | 3070（8GB） | 5070Ti（12GB） |

## 推荐工具链

- **Ollama**（最简单）：一行命令即可运行 LLM，如 `ollama run qwen2.5:14b-instruct-q4_K_M`
- **LM Studio**（带图形界面）：可视化下载和对话，兼容 OpenAI API
- **ComfyUI**（文生图/视频统一平台）：节点式工作流，灵活强大
- **Fooocus**（极简文生图）：类似 Midjourney 体验，开箱即用
- **llama.cpp + GGUF**（最灵活）：精细控制，配置稍复杂
