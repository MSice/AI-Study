## Q1：能说出至少 5 个主流 LLM 及其特点

> 详细对比见 → [主流大模型系列对比](./主流大模型系列对比.md)

| 模型系列 | 公司 | 开源/闭源 | 核心特点 |
|---------|------|----------|---------|
| **GPT** | OpenAI | 闭源 | 全能选手，生态最成熟，API 最稳定，Plugin/Function Calling 支持最好 |
| **Claude** | Anthropic | 闭源 | 长文本之王（200K token），代码和写作质量高，安全对齐好 |
| **Gemini** | Google | 闭源 | 多模态原生（文本/图片/视频/音频混合输入），和 Google 生态深度绑定 |
| **Llama** | Meta | 开源 | 开源界标杆，社区生态最大，可下载到本地自行微调 |
| **Qwen/通义千问** | 阿里 | 开源 | 中文最强开源模型，Apache 2.0 协议，从 0.5B 到 72B 全尺寸覆盖 |
| **DeepSeek** | 幻方量化 | 开源 | 性价比之王，推理/数学/代码极强，MoE 架构创新，API 价格极低 |

### 补充要点

- 同一系列通常有大/中/小多个版本，小模型快且便宜，大模型强但贵
- 这些模型之间的差距正在**快速缩小**，架构设计应做到"模型无关"，方便随时切换

## Q2：什么场景该用闭源 API？什么场景该用本地开源模型？

> 详细决策树见 → [主流大模型系列对比 - 怎么选](./主流大模型系列对比.md#怎么选决策树)

### 用闭源 API 的场景

| 场景 | 推荐模型 | 原因 |
|------|---------|------|
| 快速出产品原型 | GPT-4o | 生态最完善，文档最全，集成最多 |
| 长文档处理（合同、论文、代码库） | Claude | 200K token 上下文，长文本理解最强 |
| 多模态处理（图片/视频分析） | Gemini | 原生多模态设计，支持混合输入 |
| 预算有限但需要强推理 | DeepSeek API | 价格比 OpenAI 便宜一个数量级 |
| 不想折腾基础设施 | 任意闭源 API | API 调用简单，无需 GPU |

### 用本地开源模型的场景

| 场景 | 推荐模型 | 原因 |
|------|---------|------|
| 数据不能出境/隐私严格 | Qwen / Llama | 数据不出门，完全可控 |
| 中文为主的业务 | Qwen | 中文理解、写作能力最强 |
| 英文为主的业务 | Llama | 英文能力强，社区生态大 |
| 数学/代码推理 | DeepSeek | 推理能力和 GPT-o1 接近 |
| 需要微调专属模型 | Llama / Qwen | 开源可微调，有私有数据可训练 |
| 高频调用、成本敏感 | 任意开源模型 | 一次部署后无 API 费用 |

### 核心决策逻辑

```
数据能不能出境？
├── 不能 → 必须本地部署开源模型
│   ├── 中文为主 → Qwen
│   ├── 英文为主 → Llama
│   └── 数学/代码推理 → DeepSeek
│
└── 可以 → 可以用闭源 API
    ├── 要最全能、生态最好 → GPT
    ├── 要长文本、代码质量 → Claude
    ├── 要多模态（图片/视频）→ Gemini
    └── 要便宜 + 强推理 → DeepSeek API
```

## Q3：如何在本地用 Ollama 运行一个开源模型？

> 详细环境搭建见 → [本地搭建环境隔离方案](./本地搭建环境隔离方案.md)
> 模型选择参考 → [本地部署模型选择](./本地部署模型选择.md)

### 三步上手

**第一步：安装 Ollama**

从 [ollama.com](https://ollama.com/) 下载安装，支持 macOS / Windows / Linux。

**第二步：一行命令运行模型**

```bash
# 中文首选（14B 参数，INT4 量化，需 12GB 显存）
ollama run qwen2.5:14b-instruct-q4_K_M

# 显存只有 8GB？用 7B 版本
ollama run qwen2.5:7b-instruct-q4_K_M

# 其他推荐模型
ollama run deepseek-r1:14b          # 推理/代码场景
ollama run llama3.1:8b-instruct-q8_0  # 英文场景
```

**第三步：开始对话**

运行命令后直接进入交互式对话，输入问题即可获得回答。按 `Ctrl+D` 退出。

### 根据显存选模型

> 详细配置对照表见 → [本地部署模型选择](./本地部署模型选择.md)

| 显存 | 推荐模型上限 | 最佳选择 |
|------|------------|---------|
| 8GB | 7B-Q4 | Qwen2.5-7B-Q4（~4.5GB） |
| 12GB | 14B-Q4 | Qwen2.5-14B-Q4（~8.5GB） |
| 16GB+ | 14B-Q8 或更大 | 可尝试更高精度量化 |

### 进阶：修改模型存储位置

```bash
# 默认存储在 ~/.ollama/models，可通过环境变量修改
export OLLAMA_MODELS=~/ai-models/llm

# Windows
set OLLAMA_MODELS=D:\ai-models\llm
```

### 进阶：作为 API 服务使用

Ollama 启动后默认在 `localhost:11434` 提供兼容 OpenAI 格式的 API：

```bash
curl http://localhost:11434/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "qwen2.5:14b-instruct-q4_K_M",
    "messages": [{"role": "user", "content": "你好"}]
  }'
```

这意味着你可以用 `openai` Python 库直接连接本地 Ollama，只需把 `base_url` 改为 `http://localhost:11434/v1`。
