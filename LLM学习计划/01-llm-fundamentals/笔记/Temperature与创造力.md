# Temperature 为什么决定"创造力"

## LLM 是怎么"选词"的

大模型生成文本的本质是**一个词一个词地预测下一个最可能的词**。模型内部会算出一个概率分布，比如输入"今天天气真"：

| 候选词 | 原始概率 |
|--------|---------|
| 好 | 60% |
| 不错 | 20% |
| 热 | 10% |
| 糟糕 | 5% |
| 奇怪 | 3% |
| 离谱 | 2% |

模型根据这个概率来"抽奖"决定输出哪个词。

## Temperature 的作用

Temperature 是在抽奖之前，**对概率分布做一次"调温"**：

- **Temperature 低（如 0.1）**：概率极度集中到得分最高的词，输出确定、保守、可预测。
- **Temperature 高（如 1.5）**：概率变得平坦，每个词都有不小的机会被选中，输出多样、意外、"有创意"。

## 为什么叫"温度"

名字来自**统计物理学的玻尔兹曼分布（Boltzmann Distribution）**：

- **温度低（冰）**：分子几乎不动，整齐排列 → 确定性高、有序
- **温度高（沸水）**：分子到处乱蹦，不可预测 → 随机性高、混乱

数学上，模型用的 softmax 公式和玻尔兹曼分布形式一模一样：

```
P(词i) = exp(logit_i / T) / Σ exp(logit_j / T)
```

`T` 就是 Temperature。所以"温度"不是随便起的比喻，而是**数学公式本身就和物理学中的温度公式同构**。

## 实际使用建议

| 场景 | 推荐 Temperature |
|------|-----------------|
| 写代码、翻译、事实性问答 | 0 ~ 0.3 |
| 写文案、头脑风暴、创意写作 | 0.7 ~ 1.0 |
| 超过 1.0 | 慎用，容易胡说八道 |

## 一句话总结

> Temperature 控制的是模型"选词时的随机程度"。温度低 = 只选最稳妥的词 = 死板但可靠；温度高 = 给冷门词更多机会 = 多样但可能跑偏。
