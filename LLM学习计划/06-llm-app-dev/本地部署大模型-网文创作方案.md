# 本地搭建大模型 — 网文创作系统部署方案

> 目标：在本地部署一套能驱动"长篇网文创作系统 v4.0"的大模型环境。
> 核心需求：中文长文本生成（单次 2000-5000 字）、长上下文记忆（≥32K tokens）、角色/剧情一致性。

## 二、推荐模型对比

| 模型 | 参数量 | 中文能力 | 长文本生成 | 上下文窗口 | Q4量化显存 | 推荐度 |
|------|--------|---------|-----------|-----------|-----------|--------|
| **Qwen3-32B** | 32B（Dense） | 原生中文，极强 | 优秀 | 128K | ~16-20GB | ★★★★★ |
| **Qwen3-30B-A3B** | 30B（MoE，3B激活） | 原生中文，极强 | 优秀 | 128K | ~19-22GB | ★★★★☆ |
| **LongWriter-Zero-32B** | 32B | 强 | 顶级（专为长文优化） | 32K | ~20GB | ★★★★★ |
| **DeepSeek-V3-32B** | 32B（蒸馏版） | 极强 | 优秀 | 128K | ~16-20GB | ★★★★☆ |
| **Qwen3-8B** | 8B | 良好 | 良好 | 128K | ~5-6GB | ★★★☆☆ |
| **Qwen3-4B** | 4B | 一般 | 一般 | 128K | ~3GB | ★★☆☆☆ |

### 模型选择建议

- **首选：Qwen3-32B**（综合最强，中文原生，128K 上下文，创作质量最高）
- **长文专精：LongWriter-Zero-32B**（专为长文本生成优化，单次可生成 32K tokens）
- **性价比：Qwen3-30B-A3B**（MoE 架构，仅 3B 参数激活，推理速度快，质量接近 32B）
- **入门/测试：Qwen3-8B**（8GB 显卡即可运行，质量可接受但不够精细）

## 四、部署步骤（以推荐方案 B 为例）

### 4.1 安装 Ollama（最简方案）

```bash
# macOS
brew install ollama

# Linux
curl -fsSL https://ollama.com/install.sh | sh

# Windows
# 从 https://ollama.com/download 下载安装包
```

### 4.2 拉取模型

```bash
# 首选：Qwen3-32B（约 20GB 下载）
ollama pull qwen3:32b

# 或者：长文本专精模型
ollama pull gurubot/longwriter-zero-32b

# 入门/测试：Qwen3-8B（约 5GB 下载）
ollama pull qwen3:8b
```

### 4.3 验证模型运行

```bash
# 简单测试
ollama run qwen3:32b "你好，请用100字描述一个玄幻世界的设定"

# 查看运行状态
ollama ps
```

### 4.4 启动 API 服务

Ollama 启动后自动提供 OpenAI 兼容的 API 服务：

```bash
# 默认监听 http://localhost:11434
# API 端点：http://localhost:11434/v1/chat/completions

# 测试 API
curl http://localhost:11434/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "qwen3:32b",
    "messages": [{"role": "user", "content": "写一个玄幻小说的开头，200字"}],
    "temperature": 0.8
  }'
```

### 4.5 对接创作系统

将 v4.0 提示词系统的各阶段提示词通过 API 发送给模型：

```python
import requests
import json

OLLAMA_URL = "http://localhost:11434/v1/chat/completions"
MODEL = "qwen3:32b"

def call_llm(system_prompt: str, user_prompt: str, temperature: float = 0.8) -> str:
    """调用本地模型"""
    response = requests.post(OLLAMA_URL, json={
        "model": MODEL,
        "messages": [
            {"role": "system", "content": system_prompt},
            {"role": "user", "content": user_prompt}
        ],
        "temperature": temperature,
        "max_tokens": 8192
    })
    return response.json()["choices"][0]["message"]["content"]

# 示例：执行阶段一（世界观生成）
worldview_prompt = open("versions/v4.0/03-五阶段核心提示词.md").read()
result = call_llm(
    system_prompt="你是一位资深网文架构师",
    user_prompt="小说类型：玄幻+系统流\n目标平台：番茄\n目标总字数：100万字\n..."
)
print(result)
```

## 五、高级部署：vLLM（高性能方案）

如果需要更高的生成速度（生产级），使用 vLLM：

```bash
# 安装 vLLM
pip install vllm

# 启动 vLLM 服务（OpenAI 兼容 API）
python -m vllm.entrypoints.openai.api_server \
  --model Qwen/Qwen3-32B \
  --quantization awq \
  --max-model-len 32768 \
  --gpu-memory-utilization 0.9 \
  --port 8000

# vLLM 的吞吐量是 Ollama 的 3 倍以上
# 适合需要批量生成多章节的场景
```

## 七、参数调优建议

### 创作阶段的参数差异

| 阶段 | temperature | top_p | max_tokens | 说明 |
|------|------------|-------|------------|------|
| 世界观/角色/关系 | 0.7-0.8 | 0.9 | 4096 | 需要创意但要结构化 |
| 大纲规划 | 0.6-0.7 | 0.85 | 4096 | 需要逻辑性，降低随机性 |
| 正文写作 | 0.8-0.9 | 0.95 | 8192 | 需要文学性和创造力 |
| 评分/检查 | 0.3-0.5 | 0.8 | 2048 | 需要严谨客观，降低温度 |
| 连贯性检查 | 0.2-0.3 | 0.7 | 2048 | 最严谨，几乎不需要创意 |

### 上下文管理

网文创作的核心挑战是上下文窗口限制。即使模型支持 128K，实际使用时建议：

```
每次调用时的上下文构成：
├── 系统提示词（约 2000 tokens）
├── 已锁定基建摘要（约 1000-2000 tokens）
├── 当前章节大纲（约 500 tokens）
├── 前3章摘要（约 1500 tokens）
├── 角色状态快照（约 500-1000 tokens）
├── 活跃关系线（约 500 tokens）
├── 待回收/待埋伏笔（约 300 tokens）
└── 预留给生成（约 4000-8000 tokens）
─────────────────────────────
总计：约 10,000-16,000 tokens（远低于 128K 上限，留足余量）
```

## 八、我的设备适配方案（ROG 笔记本 / 32GB 内存 / RTX 5070 Ti）

### 硬件分析

| 项目 | 规格 | 评估 |
|------|------|------|
| **GPU** | RTX 5070 Ti Laptop（12GB GDDR7，Blackwell 架构） | 12GB 显存是关键瓶颈 |
| **内存** | 32GB | 可用于 CPU offload 扩展 |
| **显存带宽** | ~672 GB/s（GDDR7） | 比上代 GDDR6 快很多，推理速度有优势 |
| **CUDA 核心** | 5888 | 计算能力充足 |

### 可运行的模型与策略

```
12GB 显存能装下什么？

✅ Qwen3-8B Q4_K_M     → 约 5-6GB 显存，完全放入 GPU，速度最快
✅ Qwen3-14B Q4_K_M    → 约 8-9GB 显存，完全放入 GPU，质量更好
✅ Qwen3-30B-A3B Q4    → 约 19-22GB，需 GPU+CPU 混合，但仅 3B 激活，速度可接受
⚠️ Qwen3-32B Q4_K_M   → 约 16-20GB，超出显存，需 GPU+CPU offload
⚠️ LongWriter-Zero-32B → 约 20GB，同上
❌ 任何 Q8 或 FP16 的 32B 模型 → 远超显存
```

### 推荐方案（按优先级排序）

**方案 1（首选）：Qwen3-14B Q4 — 纯 GPU 推理**

```bash
ollama pull qwen3:14b
ollama run qwen3:14b
```

- 显存占用：约 8-9GB，完全放入 12GB 显存
- 推理速度：约 25-35 tokens/s
- 每章（2500字）生成时间：约 1.5-2.5 分钟
- 创作质量：良好，能胜任大部分创作任务
- **优势：速度快、稳定、不吃内存**

**方案 2（质量优先）：Qwen3-30B-A3B Q4 — MoE 架构**

```bash
ollama pull qwen3:30b-a3b
ollama run qwen3:30b-a3b
```

- 显存占用：模型文件约 19-22GB，需 GPU+CPU offload
- 实际推理：MoE 架构每次只激活 3B 参数，推理负载轻
- 推理速度：约 10-20 tokens/s（部分层在 CPU 上会拖慢）
- 每章生成时间：约 3-5 分钟
- 创作质量：接近 32B Dense 模型
- **优势：质量高，MoE 架构天然适合显存不够的场景**

**方案 3（极致质量）：Qwen3-32B Q4 — GPU+CPU 混合推理**

```bash
# 使用 llama.cpp 精确控制 GPU/CPU 层分配
# 将约 60% 的层放 GPU（~12GB），其余放内存

# 安装 llama.cpp（带 CUDA 支持）
cmake -B build -DGGML_CUDA=ON
cmake --build build --config Release

# 运行，--ngl 控制放入 GPU 的层数
./build/bin/llama-server \
  -m qwen3-32b-q4_k_m.gguf \
  --ngl 35 \
  --ctx-size 16384 \
  --host 0.0.0.0 --port 8080
```

- GPU 放 35 层（约 10-11GB），剩余层放 32GB 内存
- 推理速度：约 8-15 tokens/s
- 每章生成时间：约 4-8 分钟
- 创作质量：最高
- **优势：质量最好，但速度最慢**

### 我的建议

```
日常创作流程：

第一阶段（基建）→ 用方案 2 或 3（质量优先，基建只生成一次）
第二阶段（开篇）→ 用方案 2 或 3（开篇决定生死，质量不能妥协）
第三阶段（连载）→ 用方案 1（批量生成多章，速度更重要）
评分/检查      → 用方案 1（评分不需要创造力，14B 足够）
```

也可以同时下载多个模型，按需切换：

```bash
ollama pull qwen3:14b         # 日常创作 + 评分
ollama pull qwen3:30b-a3b     # 基建 + 开篇（质量优先时切换）
```

### 笔记本使用注意事项

1. **散热**：长时间推理会让 GPU 持续满载，确保散热模式开到最高（ROG 一般有"增强"或"极速"模式）
2. **电源**：必须插电使用，电池模式下 GPU 功耗会被限制，速度大打折扣
3. **内存占用**：跑 30B 模型时会吃掉大量内存，建议关闭其他大型应用
4. **显存监控**：用 `nvidia-smi` 实时查看显存占用，避免 OOM

```bash
# 实时监控 GPU 状态（每秒刷新）
watch -n 1 nvidia-smi
```

## 九、最低规格一句话总结

| 目标 | 最低配置 | 模型 | 体验 |
|------|---------|------|------|
| **能跑就行** | 16GB 内存 + RTX 3060 12GB | Qwen3-8B Q4 | 质量一般，速度尚可 |
| **质量可用** | 32GB 内存 + RTX 5070 Ti 12GB | Qwen3-14B Q4 或 30B-A3B Q4 | 质量良好，速度流畅 |
| **流畅创作** | 64GB 内存 + RTX 4090 24GB | Qwen3-32B Q4 | 质量高，速度流畅 |
| **无GPU** | 96GB 内存 + 高端CPU | Qwen3-32B Q4（llama.cpp） | 质量高，速度慢 |

**你的 ROG（32GB + 5070 Ti 12GB）定位**：处于"质量可用"档位，首选 Qwen3-14B 纯 GPU 推理（快且稳），需要高质量时切换 Qwen3-30B-A3B（MoE 架构，显存友好）。完全够用。
