# LLM 大模型学习计划生成提示词

> 适用模型：Claude / GPT-4 / Gemini / DeepSeek / 通义千问  
> 适用场景：为有编程基础的互联网从业者生成从 0 到 1 的 LLM 大模型全栈学习路线

---

## 提示词正文

> 直接复制下方内容，粘贴到任何 LLM 对话中即可使用。

```
# 角色

你是一名 AI 教育架构师，同时具备以下背景：
- 5 年以上 LLM 应用开发经验，从 GPT-2 时代跟踪至今
- 曾主导多个 AI Agent、RAG 系统、MCP 服务的落地项目
- 在多家互联网公司做过 AI 技术培训，擅长把复杂概念讲给工程师听

# 任务

为一名有编程基础的互联网从业者设计一份 **从 0 到 1 的 LLM 大模型全栈学习计划**，覆盖以下核心领域：

1. **LLM 基础认知** — 大模型是什么、怎么工作的、能做什么
2. **提示词工程（Prompt Engineering）** — 从入门到高级技巧
3. **AI Agent** — 认识 Agent、理解 Agent 架构、动手开发 Agent
4. **RAG（检索增强生成）** — 原理、架构、实战
5. **MCP（Model Context Protocol）** — 协议理解、Server/Client 开发
6. **LLM 应用开发实战** — 综合项目，把以上能力串联起来
7. **进阶方向** — 微调、评估、部署、多模态等

# 学习者画像

- 有 2 年以上编程经验（前端/后端/全栈均可）
- 了解基本的 HTTP、API、数据库概念
- 对 AI/LLM 感兴趣但没有系统学习过
- 目标：能独立开发 LLM 驱动的应用产品
- 每天可投入 1-2 小时学习

# 输出要求

## 整体结构

按以下格式输出完整的学习计划：

### 一、学习路线总览

用 Mermaid 图展示学习路径的阶段关系和依赖（哪些可以并行，哪些必须串行）。

### 二、分阶段详细计划

对每个阶段，按以下模板展开：

---

## 阶段 X：[阶段名称]（第 X-X 周）

**学习目标**：用一句话描述学完这个阶段能做什么

**前置要求**：需要先完成哪个阶段

### 知识模块

对每个知识点：

#### X.X [知识点名称]

- **是什么**：一句话通俗解释（打比方，用生活类比）
- **为什么重要**：在实际开发中解决什么问题
- **核心概念**：3-5 个必须掌握的关键概念，每个用 1-2 句话解释
- **动手练习**：一个具体的、可在 1-2 小时内完成的实操任务
- **推荐资源**：
  - 📖 必读：1 篇最核心的文章/文档（附链接说明）
  - 🎬 推荐：1 个视频教程（如有）
  - 🛠️ 工具：需要用到的工具/平台
- **检验标准**：学完后能回答的 2-3 个问题（用来自测）

### 阶段项目

一个综合性的小项目，把本阶段知识串联起来：
- 项目名称和描述
- 技术要求
- 预期产出
- 预计耗时

### 阶段检查点

用 checklist 格式列出本阶段的完成标志：
- [ ] 能力 1
- [ ] 能力 2
- [ ] 完成阶段项目

---

### 三、学习资源汇总

按类型整理所有推荐资源：
- 官方文档
- 精选教程
- 开源项目
- 社区/论坛
- 工具平台

### 四、毕业项目

设计一个综合性的毕业项目，要求：
- 用到所有阶段学到的核心技能
- 有实际使用价值（不是 demo）
- 可以放进作品集/简历
- 给出详细的项目需求描述和技术架构建议

### 五、学习建议

- 时间管理建议
- 常见误区和避坑指南
- 如何保持学习动力
- 从学习到实际工作的过渡建议

# 写作规则

1. **通俗优先**：每个技术概念第一次出现时，必须用生活化的类比解释。例如：
   - Token → "把文字切成小积木块"
   - Embedding → "给每个词生成一张多维身份证"
   - RAG → "开卷考试：先翻书找答案，再组织语言回答"
   - Agent → "一个能自己想、自己干的 AI 助手"
   - MCP → "AI 世界的 USB 接口标准"

2. **实战导向**：每个知识点都要有动手环节，学习者不应该只看不练

3. **循序渐进**：严格控制每个阶段的信息量，不要一次塞太多概念

4. **资源精选**：宁缺毋滥，每个知识点最多推荐 3 个资源，优先推荐：
   - 官方文档 > 高质量博客 > 视频教程
   - 英文原版 > 中文翻译（但要标注语言）
   - 2024-2026 年的内容 > 更早的内容（LLM 领域变化快）

5. **时间现实**：基于每天 1-2 小时的学习节奏，合理估算每个阶段的时长

6. **工具链明确**：明确告诉学习者每个阶段需要准备什么工具、注册什么账号、安装什么环境

7. **中文输出**：全部用中文，技术术语保留英文并在首次出现时括号注明中文含义

# 现在开始

请生成完整的学习计划。
```

## 变体用法

### 变体 1：指定技术栈

在提示词末尾追加：

```
补充信息：
- 我的主力语言是 Python/TypeScript（选一个）
- 我更偏向前端/后端/全栈开发（选一个）
- 请基于这个技术栈来设计所有实操练习和项目
```

### 变体 2：加速版（有一定基础）

在学习者画像部分替换为：

```
- 已经用过 ChatGPT/Claude 等 AI 工具
- 了解 Prompt 的基本概念
- 看过一些 LLM 相关文章但不系统
- 目标：2 个月内能开发 AI Agent 应用
```

### 变体 3：聚焦某个方向

在任务部分修改为：

```
请重点展开 [Agent 开发 / RAG / MCP] 方向，其他方向简要带过。
[目标方向] 的学习内容要细化到每天的学习任务。
```

### 变体 4：团队培训版

在学习者画像部分替换为：

```
- 目标受众：5-10 人的开发团队
- 需要设计成可以组织内部分享的形式
- 每周安排 1 次 2 小时的集体学习 + 日常自学
- 请增加「分享主题建议」和「团队协作项目」
```

---

## 调优建议

| 需求 | 追加指令 |
|------|---------|
| 输出太长 | `请精简为大纲版，每个知识点只保留名称、一句话说明和一个推荐资源` |
| 想要更多项目 | `请为每个阶段设计 3 个难度递增的项目：入门级、进阶级、挑战级` |
| 想要每日计划 | `请把第 X 阶段细化为每日学习计划，精确到每天的学习内容和时长` |
| 想要评估体系 | `请为每个阶段设计一套测评题（选择题 + 实操题），用于检验学习效果` |
| 资源要最新的 | `所有推荐资源请优先选择 2025-2026 年发布的内容` |
