# 00 — 平台适配与字数分段策略

> 不同平台的算法逻辑、读者画像、付费模式截然不同，同一个故事在番茄和起点需要完全不同的写法。
> 不同字数量级的小说在结构、节奏、角色数量上也有本质差异。

---

## 一、平台适配策略

### 番茄小说（免费流量模式）

```yaml
platform: "fanqie"
model: "免费阅读 + 广告分成"
reader_profile:
  age: "18-35岁为主，下沉市场占比高"
  reading_habit: "碎片化阅读，通勤/睡前场景"
  patience: "极低，首章30秒内不抓人就划走"
  preference: "强情绪、快节奏、高密度爽点"

writing_rules:
  chapter_length: "2000-2300字（最优区间）"
  hook_density: "每2000字至少1个反转或爽点"
  golden_chapters: "前3章决定生死，首章完读率需>15%"
  update_frequency: "日更4000-6000字（低于4000算法不推流）"
  
  opening_strategy:
    - "第一句话就要有冲突或悬念，禁止环境描写开头"
    - "前500字内必须出现核心矛盾"
    - "前3章完成：主角出场 + 金手指展示 + 第一个小爽点"
    - "每章结尾必须有钩子，逼读者点下一章"

  content_strategy:
    - "情绪价值 > 文学价值，核心是持续供给情绪"
    - "每3-5千字一个小高潮，每2-3万字一个大打脸"
    - "对话占比40-50%，减少大段描写"
    - "章节标题要有悬念感或冲突感"

  monetization_milestones:
    - "2万字：申请签约"
    - "8万字：完成验证期（日均阅读>500人）"
    - "10万字：首秀推荐"
    - "14万字：建议存稿量"

  banned_topics:
    - "穿越民国、官场斗争等敏感题材"
    - "过度血腥暴力、涉政内容"

  hot_formulas:
    male: "系统+身份反差+冲突（如《战神系统：废柴村签到十年》）"
    female: "重生+反套路+情感冲突（如《病娇太子妃：重生后我黑化了》）"
    cross: "奇幻+脑洞、末世+萌宝、悬疑+直播等跨类型融合"
```

### 起点中文网（付费订阅模式）

```yaml
platform: "qidian"
model: "前N章免费 + 后续付费订阅"
reader_profile:
  age: "20-40岁，网文老读者占比高"
  reading_habit: "长线追更，愿意为好故事付费"
  patience: "中等，能接受铺垫但前3章仍是生死线"
  preference: "复杂世界观、深度人设、长线叙事、创新设定"

writing_rules:
  chapter_length: "2500-4000字"
  hook_density: "每5000字至少1个小高潮"
  golden_chapters: "前3章3000字内完成'主角危机+金手指+悬念'三要素"
  update_frequency: "日更4000-8000字（稳定更新比爆更重要）"

  opening_strategy:
    - "开篇前几十个字决定点击率"
    - "允许适度铺垫，但前3章必须亮出核心卖点"
    - "采用'单元剧+主线'结构：5-10章完成一个独立事件，同时推进长线"
    - "投稿三件套：大纲(800-1000字) + 前三章正文(6000字) + 人设小传"

  content_strategy:
    - "世界观创新是核心竞争力（赛博修仙、量子武道等跨界融合）"
    - "人设要有多面性和缺陷，避免龙傲天"
    - "叙事减脂：删除所有打断体验的冗余文字"
    - "细节必须服务于剧情和人物刻画，无关描写一律删除"
    - "真实感极致追求，细节经得起推敲"

  quality_standards:
    - "力量体系要有独特的内在逻辑"
    - "社会结构与修炼体系匹配"
    - "反套路叙事受欢迎（苟道流、黑化复仇流）"
    - "允许哲学思辨与存在主义探讨"

  paid_chapter_strategy:
    - "付费章节前的最后一个免费章必须是大悬念"
    - "付费后的第一章必须立刻给出回报（不能让付费读者失望）"
    - "卡点设置：在关键反转/揭秘前切章"
```

### 书旗小说 / 七猫小说（免费+广告模式）

```yaml
platform: "shuqi_qimao"
model: "免费阅读 + 广告 + 金币激励"
reader_profile:
  age: "25-45岁，下沉市场为主"
  reading_habit: "听书占比高，碎片化消费"
  patience: "低，类似番茄"
  preference: "现代言情、古代言情、幻想言情、都市"

writing_rules:
  chapter_length: "1800-2500字"
  hook_density: "每章至少1个情绪高点"
  update_frequency: "日更3000-5000字"

  opening_strategy:
    - "与番茄类似，快节奏开局"
    - "女频占比更高，言情元素前置"
    - "前3章建立CP感或核心矛盾"

  content_strategy:
    - "适合听书的写法：对话多、短句多、场景转换清晰"
    - "章节之间衔接要明确（听书用户可能中断后继续）"
    - "情感线要清晰直白，暗线不宜过多"
    - "现言甜宠+职业元素是当前热门组合"

  special_notes:
    - "七猫金币系统鼓励用户长时间阅读，章节不宜过长"
    - "听书功能要求文字节奏适合朗读，避免过于书面化的表达"
```

### 知乎盐选（短篇精品模式）

```yaml
platform: "zhihu_yanxuan"
model: "会员订阅 + 单篇付费"
reader_profile:
  age: "22-35岁，高知群体"
  reading_habit: "追求质量而非数量，偏好短篇精品"
  patience: "较高，但对逻辑和文笔要求严格"
  preference: "职业探秘、社会议题、悬疑推理、反转短篇"

writing_rules:
  chapter_length: "3000-5000字"
  total_length: "1-10万字为主"
  
  opening_strategy:
    - "可以用更文学化的开头，但前500字必须建立好奇心"
    - "适合倒叙、悬念式开篇"
    - "第一人称叙述接受度高"

  content_strategy:
    - "文笔要求高于其他平台"
    - "逻辑严密性是核心，读者会挑BUG"
    - "社会洞察和职业细节是加分项"
    - "结尾反转是标配，但要意料之外情理之中"
    - "适合规则怪谈、悬疑推理、现实主义题材"
```

## 二、字数分段体系

### 短篇档（5-20万字）

```yaml
tier: "short"
range: "5-20万字"
chapters: "25-100章"
suitable_platforms: ["知乎盐选", "番茄短篇", "七猫"]
suitable_genres: ["悬疑推理", "规则怪谈", "短篇言情", "都市逆袭"]

structure:
  volumes: "1-2卷，不分卷也可"
  character_count:
    protagonist: 1
    core_supporting: "2-4"
    important_supporting: "3-6"
    extras: "少量"
  
  pacing:
    thrill_cycles: "2-4个潮汐周期"
    cycle_length: "5-10章/周期"
    buildup_ratio: "15%"
    suppression_ratio: "25%"
    climax_ratio: "30%"
    resolution_ratio: "30%"
  
  plot_density: "高密度，无废章"
  foreshadowing: "伏笔跨度短（5-15章内回收），不超过3条暗线"
  
  golden_rule: "短篇的核心是'一个精彩的点子讲透'，不要贪多"

  opening:
    - "第1章直接进入核心事件"
    - "前3章完成所有主要角色登场"
    - "前5章必须让读者明确'这个故事要讲什么'"
  
  ending:
    - "结局要干脆利落，不拖泥带水"
    - "最后1-2章集中收束所有线索"
    - "允许开放式结局，但核心矛盾必须解决"
```

### 中篇档（20-60万字）

```yaml
tier: "medium"
range: "20-60万字"
chapters: "100-300章"
suitable_platforms: ["番茄", "起点", "书旗", "七猫"]
suitable_genres: ["都市重生", "甜宠言情", "系统流", "末世生存"]

structure:
  volumes: "3-5卷"
  character_count:
    protagonist: "1-2"
    core_supporting: "4-8"
    important_supporting: "8-15"
    extras: "适量"
  
  pacing:
    thrill_cycles: "5-10个潮汐周期"
    cycle_length: "8-15章/周期"
    suppression_min: "4章"
    climax_min: "2章"
  
  plot_density: "中高密度，允许少量过渡章但必须有信息量"
  foreshadowing: "伏笔跨度中等（10-30章回收），可维持5-8条暗线"
  
  golden_rule: "中篇的核心是'一条清晰的成长线 + 2-3个精彩的大高潮'"

  milestone_chapters:
    - "第1-3章：黄金三章（生死线）"
    - "第10章左右：第一个大爽点（留住读者）"
    - "第30章左右：中期转折（打开新格局）"
    - "60%处：最大危机（全书最低谷）"
    - "80%处：终极反转"
    - "最后10%：大结局 + 收尾"
```

### 长篇档（60-150万字）

```yaml
tier: "long"
range: "60-150万字"
chapters: "300-750章"
suitable_platforms: ["起点", "番茄", "书旗"]
suitable_genres: ["玄幻", "仙侠", "科幻", "历史架空", "宫斗权谋"]

structure:
  volumes: "5-10卷"
  character_count:
    protagonist: "1-2"
    core_supporting: "8-12"
    important_supporting: "15-25"
    extras: "大量（需要角色管理系统）"
  
  pacing:
    thrill_cycles: "10-20个潮汐周期"
    cycle_length: "10-20章/周期"
    suppression_min: "5章"
    climax_min: "3章"
    volume_climax: "每卷结尾必须有超级高潮"
  
  plot_density: "中密度，允许日常章但占比不超过20%"
  foreshadowing: "伏笔跨度长（可跨卷回收），可维持10-15条暗线"
  subplot_count: "2-4条副线，与主线交织"
  
  golden_rule: "长篇的核心是'不断扩大的世界 + 不断升级的对手 + 不断成长的主角'"

  anti_fatigue:
    - "每3-5卷更换主要活动场景，避免审美疲劳"
    - "每2卷引入至少1个重要新角色"
    - "中期必须有一次'格局升级'（从个人恩怨到势力对抗到世界级冲突）"
    - "副线角色要有独立的精彩剧情，不能只是主角的工具人"
    - "允许阶段性告别某些角色，后期回归制造惊喜"
```

### 超长篇档（150万字以上）

```yaml
tier: "ultra_long"
range: "150万字+"
chapters: "750章+"
suitable_platforms: ["起点"]
suitable_genres: ["玄幻", "仙侠", "科幻史诗"]

structure:
  volumes: "10卷+"
  character_count:
    protagonist: "1-2（主角必须有足够的成长空间）"
    core_supporting: "12-20"
    important_supporting: "25-40"
    extras: "海量（必须有角色数据库管理）"
  
  pacing:
    thrill_cycles: "20+个潮汐周期"
    mega_arcs: "3-5个大弧（每个大弧包含多个潮汐周期）"
    arc_escalation: "每个大弧的格局必须比上一个更大"
  
  world_expansion:
    - "世界观必须有足够的纵深支撑150万字以上的探索"
    - "建议采用'洋葱模型'：每揭开一层，发现更大的世界"
    - "至少3次'世界观颠覆'（读者以为了解了世界，然后发现只是冰山一角）"
  
  character_management:
    - "建立角色数据库，每50章做一次角色状态审计"
    - "退场角色要有合理交代，不能人间蒸发"
    - "每个大弧有独立的'角色群'，避免同时活跃角色过多"
    - "主角的性格必须有阶段性变化，不能从头到尾一个样"
  
  reader_retention:
    - "每100章做一次'回顾章'，帮读者梳理剧情"
    - "长期伏笔回收时要有'回忆闪回'，提醒读者之前的铺垫"
    - "每个大弧结束后有'间章'，展示世界其他角落的变化"
    - "定期引入新鲜元素（新能力、新阵营、新规则）防止疲劳"
  
  golden_rule: "超长篇的核心是'史诗感'——让读者觉得在见证一个世界的兴衰"
```

## 三、跨类型融合指南

当小说涉及多种类型时，需要确定**主引擎**和**副引擎**：

```yaml
fusion_rules:
  primary_engine: "决定核心写作逻辑和节奏的引擎（只能1个）"
  secondary_engines: "提供辅助策略的引擎（1-2个）"
  
  conflict_resolution:
    - "当两个引擎的规则冲突时，以主引擎为准"
    - "副引擎的规则作为'加分项'而非'必须项'"
  
  popular_combos:
    - primary: "system_innovation"
      secondary: ["thrill_curve"]
      example: "系统流爽文（系统创新为核心，爽感节奏为辅助）"
    
    - primary: "romance"
      secondary: ["rule_horror"]
      example: "言情+悬疑（感情线为核心，悬疑推理增加深度）"
    
    - primary: "imagination_first"
      secondary: ["thrill_curve", "system_innovation"]
      example: "玄幻+系统+爽文（世界观创新为核心，系统和爽感为辅助）"
    
    - primary: "urban_reborn"
      secondary: ["thrill_curve"]
      example: "都市重生爽文（重生逆袭为核心，爽感节奏为辅助）"
    
    - primary: "historical"
      secondary: ["romance"]
      example: "历史架空+言情（历史推演为核心，感情线为辅助）"
    
    - primary: "rule_horror"
      secondary: ["mystery"]
      example: "规则怪谈+推理（规则体系为核心，推理解谜为辅助）"
```
