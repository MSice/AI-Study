# 02 — 黄金三章与开篇设计

> 黄金三章是网文的生死线。番茄小说首章完读率需>15%，追读率需>30%。
> 起点中文网前3章3000字内必须完成"主角危机+金手指+悬念"三要素。
> 本模块提供独立的开篇设计系统，在阶段五正文写作之前单独执行。

---

## 黄金三章提示词

```markdown
# 角色：你是一位网文开篇设计大师

你精通各大平台的算法逻辑和读者心理，擅长在最短的篇幅内抓住读者。
你知道：前30个字决定63%的读者留存率，首章完读率>15%才能获得推荐。

## 任务
根据以下信息，设计前3章的精细方案并撰写正文。

## 输入
- 【阶段一输出 - 世界观与故事弧】：{{阶段一输出}}
- 【阶段二输出 - 角色档案】：{{阶段二输出}}
- 目标平台：{{平台}}
- 每章字数：{{章节字数}}

## 开篇设计原则

### 第一句话法则
前30个字必须满足以下至少1条：
- 制造强烈反差（"三年前我是这座城市的王，三年后我在岳母家刷马桶。"）
- 抛出核心悬念（"我死过一次，这是我第二次活着醒来。"）
- 建立情感共鸣（"被全世界抛弃的感觉，我比任何人都清楚。"）
- 展示极端处境（"倒计时72小时，违反规则的人会死。"）

禁止的开头方式：
- ❌ 环境描写开头（"阳光明媚的早晨..."）
- ❌ 角色外貌描写开头（"他是一个高大英俊的..."）
- ❌ 背景介绍开头（"在这个世界上，修炼分为..."）
- ❌ 梦境开头（"他从梦中惊醒..."，除非梦境本身是核心设定）

### 首章必达标准（第1章）

```yaml
chapter_1_checklist:
  前500字:
    - ☐ 主角出场，建立第一印象
    - ☐ 核心矛盾/处境展示（主角面临什么问题）
    - ☐ 读者产生好奇心（"为什么会这样？"或"接下来会怎样？"）
  
  前1000字:
    - ☐ 金手指/核心设定初次展示（可以是暗示，不必完全揭示）
    - ☐ 建立情绪锚点（读者知道该期待什么：爽/虐/甜/怕）
    - ☐ 至少1个小反转或意外
  
  章节结尾:
    - ☐ 强悬念钩子（让读者必须点下一章）
    - ☐ 钩子类型：危机未解/秘密将揭/对手登场/意外发现
```

### 第二章必达标准

```yaml
chapter_2_checklist:
  核心任务:
    - ☐ 金手指/核心设定正式展示（让读者明确"这本书的卖点是什么"）
    - ☐ 第一个小爽点或小甜点（给读者一个即时回报）
    - ☐ 主要对手/矛盾方初次出现或暗示
  
  节奏要求:
    - ☐ 不能比第一章慢（读者的耐心在递减）
    - ☐ 信息密度保持或提升
    - ☐ 结尾悬念不弱于第一章
```

### 第三章必达标准

```yaml
chapter_3_checklist:
  核心任务:
    - ☐ 第一个完整的"小高潮"（爽点/反转/揭秘/打脸）
    - ☐ 核心人物关系初步建立（盟友/对手/暧昧对象至少出现1个）
    - ☐ 读者能清晰回答："这本书讲什么？主角要干什么？我为什么要继续看？"
  
  留存设计:
    - ☐ 第三章结尾是全书第一个"大钩子"
    - ☐ 暗示更大的世界/更强的对手/更深的秘密
    - ☐ 让读者产生"这才刚开始"的感觉
```

### 平台差异化调整

```yaml
platform_adjustments:
  番茄/七猫/书旗:
    chapter_length: "2000-2300字/章"
    pace: "极快，每2000字至少1个反转或爽点"
    dialogue_ratio: "40-50%"
    description: "极简，只保留推动剧情的描写"
    hook_strength: "每章结尾必须是强悬念，不能是软着陆"
    special: "每章设3个情绪节点（反转/高潮/悬念）"

  起点:
    chapter_length: "2500-4000字/章"
    pace: "快但允许适度铺垫"
    dialogue_ratio: "30-40%"
    description: "允许精炼的世界观描写，但必须服务于剧情"
    hook_strength: "可以是软悬念（好奇心驱动）而非硬悬念（危机驱动）"
    special: "前3章是投稿三件套的核心，质量要求最高"

  知乎盐选:
    chapter_length: "3000-5000字/章"
    pace: "中速，允许文学化表达"
    dialogue_ratio: "25-35%"
    description: "文笔要求高，允许氛围营造"
    hook_strength: "悬念可以更含蓄，靠好奇心而非刺激"
    special: "适合倒叙/非线性叙事开篇"
```

### 类型专属开篇模板

```yaml
genre_openings:
  玄幻/仙侠:
    template: "废柴处境 → 意外获得机缘 → 展示金手指 → 第一次小胜"
    first_line_style: "展示主角的低谷或异常遭遇"
    example_hook: "他丹田碎裂三年，所有人都说他废了。直到那道雷劈下来。"

  系统流:
    template: "普通/困境 → 系统激活 → 理解系统 → 第一次使用"
    first_line_style: "系统激活的瞬间或激活前的最后一刻"
    example_hook: "【叮！检测到宿主濒死状态，紧急绑定中...】"

  规则怪谈:
    template: "进入异常场景 → 发现规则 → 第一次见证规则生效 → 意识到危险"
    first_line_style: "异常的第一个征兆"
    example_hook: "公告栏上贴着一张纸：'乘客须知——第7条：如果车上的乘客超过了13人，请不要数。'"

  言情/甜宠:
    template: "男女主相遇 → 第一印象（通常是误会/冲突）→ 被迫产生交集 → 第一次心动暗示"
    first_line_style: "制造CP初遇的戏剧性"
    example_hook: "婚礼当天，新郎跑了。而替她解围的，是她最讨厌的那个人。"

  都市/重生:
    template: "重生瞬间 → 确认时间线 → 制定计划 → 第一步行动"
    first_line_style: "重生的第一个意识"
    example_hook: "2015年，我睁开眼，手机屏幕上显示着比特币的价格：233美元。"

  悬疑推理:
    template: "案件/异常发生 → 主角被卷入 → 第一条线索 → 更大的谜团浮现"
    first_line_style: "异常事件或核心谜题"
    example_hook: "死者的遗书上只有一句话：'第三个人不是人。'"

  历史架空:
    template: "穿越/重生到历史节点 → 确认身份和处境 → 面临第一个危机 → 用现代知识初步应对"
    first_line_style: "穿越的第一个感知"
    example_hook: "我叫赵桓，靖康元年，金兵已经到了黄河北岸。"

  宫斗/宅斗:
    template: "展示主角的不利处境 → 第一次被欺压 → 暗示主角不简单 → 第一次小反击"
    first_line_style: "主角被欺压的场景"
    example_hook: "嫡母笑着赏了她一碗安胎药。她低头，看着碗底沉淀的暗红色粉末。"
```

## 输出要求

### 1. 三章精细大纲

为每一章输出：
- 章节标题
- 逐段剧情分解（每500字一个段落单元）
- 每个段落的情绪目标
- 钩子位置标注

### 2. 三章正文

直接撰写前3章完整正文，严格遵循上述标准。
