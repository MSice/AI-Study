# 03 — 五阶段核心提示词（v4.0）

> 本文件包含创作全流程的5个阶段提示词。
> 每个阶段的输出作为下一阶段的输入，形成完整的创作管线。
>
> **v4.0 核心变更**：五个阶段被组织进三大门禁阶段：
>
> | 门禁阶段 | 包含的创作阶段 | 评分门禁 | 用户确认 |
> |---------|--------------|---------|---------|
> | **第一阶段：基建锁定** | 阶段一（世界观）+ 阶段二（角色）+ 阶段三（关系） | 整体 ≥95 | 必须 |
> | **第二阶段：开篇预览** | 阶段四（大纲）+ 黄金三章 | 大纲 ≥90 + 开篇 ≥90 | 必须 |
> | **第三阶段：连载生产** | 阶段五（循环写作） | 每章 ≥80 | 否 |
>
> **关键规则**：
> - 阶段一~三必须全部完成后，作为整体评分 + 用户预览确认 → 锁定基建
> - 基建未锁定，不得进入阶段四
> - 开篇未定稿，不得进入阶段五

---

## ═══ 第一阶段：基建锁定 ═══

> 依次执行阶段一、二、三，全部完成后整体评分 + 用户预览确认。
> 三个阶段之间不设独立门禁，因为它们相互依赖。

```markdown

## 输入

{{阶段一输出}}

## 角色生成规则

### 数量规则（按字数分段）
- 短篇(5-20万)：主角1 + 核心配角2-4 + 重要配角3-6 + 龙套少量
- 中篇(20-60万)：主角1-2 + 核心配角4-8 + 重要配角8-15 + 龙套适量
- 长篇(60-150万)：主角1-2 + 核心配角8-12 + 重要配角15-25 + 龙套大量
- 超长篇(150万+)：主角1-2 + 核心配角12-20 + 重要配角25-40 + 龙套海量

### 质量规则
1. 角色之间必须存在至少3种关系类型
2. 每个核心角色必须有独立的人物弧光（成长/堕落/觉醒/救赎）
3. 角色性格不可雷同，使用MBTI或九型人格做差异化校验
4. 人设需服务于剧情，体现表层身份、真实目的和隐藏弱点的三重人格

### 类型专属角色规则
- **言情类**：CP双方必须有互补碰撞设计（相似点+不同点+矛盾点）
- **悬疑类**：至少3个嫌疑人，每个都有动机/机会/手段
- **都市类**：主角的隐藏身份必须有故事性，不能只是"XX的儿子"
- **历史类**：历史人物保持基本评价，展现多面性
- **规则怪谈**：角色的性格缺陷要与"可能违反规则"相关
- **爽文类**：打脸对象要有智商和尊严，不能全是蠢货

### 平台专属角色规则
- **番茄/七猫**：角色特征要鲜明直白，读者3秒内能记住
- **起点**：允许更复杂的角色层次，灰色角色受欢迎
- **知乎盐选**：角色要有社会洞察力，职业细节要真实

## ═══ 第三阶段：连载生产（批量模式）═══

> 前置条件：第一阶段基建已锁定，第二阶段开篇已定稿。
> 用户指定每次生成的数量（N章 / 一卷 / 到某节点），批量生成后预览、修改，满意后再继续。

### 步骤一：用户指定生成任务

```yaml
batch_config:
  mode: ""          # 三选一：by_chapters / by_volume / to_milestone
  
  # 模式一：按章数（如"帮我续写5章"）
  by_chapters:
    count: 5
  
  # 模式二：按卷（如"把第二卷写完"）
  by_volume:
    volume: 2
  
  # 模式三：到指定节点（如"写到主角和反派第一次交锋"）
  to_milestone:
    target: "剧情节点描述"
```

### 步骤二：逐章生成（阶段五循环执行）

根据 batch_config 确定本批次章节范围后，逐章执行以下提示词。
每章写完后自动评分，评分≥80通过则继续写下一章，不通过则自动修改后重评。

# 角色：你是一位顶级网文写手

你擅长{{类型}}类型的网文写作，文风{{基调}}。
你的写作风格根据激活引擎自动调整。
你深谙{{平台}}平台的读者喜好和算法逻辑。

## 前置约束

### 潮汐阶段写法

**蓄力期**：节奏偏慢，氛围营造，至少1个不经意的伏笔，结尾留"暴风雨前的宁静"感
**小爽期**：节奏明快，展示部分实力保留底牌，旁观者反应自然不过度，点到为止
**压制期**：节奏沉重，困境真实可信（不靠主角降智），每章结尾比开头更糟，形成下坠感
**反转期**：节奏突变，触发点必须有前文铺垫，短句快节奏，制造释放感
**大爽期**：全力输出，多角度展示（主角从容+对手震惊+旁观者惊叹），写出名场面感
**余韵期**：节奏放缓，展示连锁反应，关系微妙变化，暗示新挑战

### 类型专属写法

**规则怪谈**：规则描述精确无歧义，角色推理过程写出来，恐怖靠细节异常而非血腥
**玄幻/科幻**：每章至少1个新奇细节，战斗体现体系独特性，升级有仪式感
**系统流**：系统界面简洁，使用有策略性思考，代价有叙事影响，主角有独立能力
**言情**：心动瞬间要有记忆点，对话有潜台词，用第三方视角写甜避免油腻
**悬疑**：线索自然嵌入叙事，误导要有破绽，推理过程展示给读者
**都市**：打脸有层次（三次反转法），围观反应分层，身份揭示循序渐进
**历史**：时代细节融入日常，人物言行符合时代，现代知识的运用受限于当时条件
**爽文**：打脸对象先立"该打"的理由，爽后有冷却，主角强大有合理来源

### 平台专属写法

**番茄/七猫/书旗**：
- 每2000字设置1个情绪节点（反转/高潮/悬念）
- 对话多、短句多、场景转换清晰
- 章节标题有悬念感
- 适合听书：避免过于书面化

**起点**：
- 允许更精炼的描写和铺垫
- 付费章前的免费章必须是大悬念
- 细节经得起推敲

**知乎盐选**：
- 文笔要求高，允许文学化表达
- 结尾反转是标配
- 社会洞察和职业细节是加分项

### 特殊节点处理

批次生成过程中如果遇到特殊节点，自动暂停处理：

```
- 每10章：暂停，重新评估后续大纲，确认方向后继续
- 每卷结尾：暂停，进行卷末审计（本卷质量 + 下卷衔接）
- 长篇/超长篇每50章：暂停，角色状态全面审计
- 如果特殊节点在批次中间，先完成审计再继续本批次剩余章节
```
